🔑 GitHub 토큰 발급 가이드

TVCF 미집행 광고주 시스템 - 작업자 전용

왜 토큰이 필요한가요?
이 토큰은 작업용 페이지에서 "집행여부" 변경 시, 그 결정사항을 GitHub에 자동 저장하기 위해 사용됩니다. 한 사람이 변경하면 팀 전체에게 자동 동기화됩니다.

  • 1부: GitHub 계정 준비 (처음 한 번)
  • 2부: 토큰 발급 (10분 소요)
  • 3부: TVCF 페이지에 토큰 입력
  • 4부: 보안 주의사항
  • FAQ

1부: GitHub 계정 준비

1GitHub 계정 만들기

아직 GitHub 계정이 없다면 https://github.com/signup에서 무료로 가입하세요. 이메일 인증 필요.

2저장소(Repo) 접근 권한 받기

관리자(동백)에게 본인 GitHub 계정 이름을 알려주세요. tvcf-monitor 저장소에 Collaborator로 초대받아야 합니다.

초대받으면 이메일이 오니까 Accept invitation 클릭!

32단계 인증 켜기 (강력 추천)

GitHub Settings → Password and authentication → Two-factor authentication 활성화. 토큰이 노출돼도 추가 보호막이 됩니다.

2부: 토큰 발급

1GitHub Settings 들어가기

로그인 후 우측 상단 프로필 아이콘 클릭 → Settings

또는 바로 가기: https://github.com/settings/profile

2Developer Settings로 이동

좌측 메뉴에서 맨 아래로 스크롤 → Developer settings 클릭

또는 바로 가기: https://github.com/settings/apps

3Personal access tokens (classic) 선택

좌측 메뉴에서:

  • Personal access tokens 클릭 (확장 메뉴)
  • Tokens (classic) 선택

바로 가기: https://github.com/settings/tokens

4새 토큰 생성

오른쪽 위 Generate new token ▼ 클릭 → Generate new token (classic) 선택

GitHub 비밀번호 다시 입력 요청 시 입력하세요.

5토큰 정보 입력

Note (메모):

TVCF 미집행 광고주 시스템

Expiration (만료):

90 days (90일)

※ 만료되면 다시 발급하면 됩니다. 보안상 무제한보다 권장.

Select scopes (권한 범위) — 가장 중요!

  • ✅ repo 체크 (전체 체크박스 하나만 누르면 하위 항목 다 선택됨)
  • 나머지는 체크하지 않습니다
⚠️ repo 체크 외에는 절대로 다른 권한 추가하지 마세요. (delete_repo, admin 등은 위험)
6토큰 생성 & 복사

맨 아래 초록색 Generate token 버튼 클릭

다음 화면에서 토큰이 ghp_xxxxxxxxxxxx... 형식으로 나타납니다.

⚠️ 이 화면에서 토큰을 복사하지 않으면 다시 볼 수 없습니다!
복사 아이콘 (📋) 클릭하거나 토큰 전체를 드래그해서 복사하세요.

복사한 토큰을 임시로 메모장 같은 곳에 붙여두세요 (다음 단계에서 사용).

3부: TVCF 페이지에 토큰 입력

1작업용 페이지 열기

https://ehdqor8212.github.io/tvcf-monitor/

2토큰 입력 버튼 클릭

페이지 하단의 🔑 영역에서 토큰 입력 버튼 클릭

3토큰 붙여넣기

모달이 열리면 복사한 토큰 (ghp_xxxxxxxxxxxx) 을 붙여넣고 저장 & 검증 클릭

4완료 확인

"✓ 토큰이 검증되었습니다" 메시지가 뜨면 성공!

이제 우측 상단 Sync: ON 표시가 보이고, 집행여부 변경 시 자동으로 GitHub에 저장됩니다.

🎉 설정 완료! 이제 너의 결정사항은 팀 전체에 자동 동기화됩니다.

4부: 보안 주의사항

  • 토큰은 절대 다른 사람에게 알려주지 마세요.
  • 토큰을 메모장, Slack, 메일 등에 평문으로 저장하지 마세요.
  • 회사 PC에서만 토큰을 입력하세요 (개인 PC, 카페 등 공용 PC 금지).
  • 의심스러운 활동이 보이면 즉시 토큰을 폐기하세요 (GitHub Settings → Tokens → Revoke).
  • 토큰이 만료되면 같은 방법으로 새로 발급하세요.
토큰이 노출되었을 때 피해 범위?
이 토큰은 tvcf-monitor 저장소에만 접근 권한이 있습니다. TVING 사내 시스템, TAS, 회사 데이터에는 영향이 없습니다. 그래도 즉시 폐기하는 것이 안전합니다.

자주 묻는 질문

Q. 토큰을 다른 PC에서도 써야 하나요?

네. 토큰은 브라우저에 저장되므로 다른 PC에서 작업하려면 그 PC에서도 한 번 입력해야 합니다. 같은 토큰을 재사용하면 됩니다.

Q. 토큰 만료되면 어떻게 되나요?

동기화가 멈춥니다. 새 토큰을 발급하고 페이지에 입력하면 즉시 복구됩니다.

Q. 토큰을 잊어버렸어요

토큰은 발급 시 한 번만 보입니다. 잊었다면 기존 토큰을 폐기(Revoke)하고 새로 발급하세요.

Q. 다른 사람의 변경 사항이 안 보여요

페이지 우측 상단의 ↻ 동기화 버튼을 클릭하거나 새로고침(Ctrl+F5) 하세요.

Q. 변경했는데 GitHub에 안 저장됐어요

Sync: ON 표시 확인 → 화면 위 알림 확인 → 콘솔(F12)에서 에러 메시지 확인 → 관리자에게 문의
